- Bundle contract validator is `backend/app/validator/bundle.py`; runtime execution is `backend/app/executor/module_runner.py`.
- Required files in bundle root: `module.py`, `metric.py`, `bundle.toml`.
- `module.py` validator requirements: at least one class inheriting a `*Signature`, at least one `dspy.Module` subclass, and top-level `build_program()`.
- `metric.py` validator requirement: top-level `judge_metric(example, prediction)` with at least two parameters.
- Runtime-required metric return shape is stricter than validator: exactly `{score, rationale, flags, raw_response}` with numeric `score`, string `rationale`, and `list[str]` flags.
- Optional bundle hooks: `build_lm()` in `module.py`; bundle metadata may include `optimized_program_state`, which runtime passes into `program.load_state(...)` if supported.
- Optimization assumes the built program exposes predictor output fields through `named_predictors()` signatures; otherwise optimization fails before compiling.
- Execution path: sample zip from `GET /samples/module-bundle`, upload/validate via module endpoints, smoke test via `run_bundle_eval`, agent run tasks via the worker, optimization via `run_bundle_optimization()` subprocess path.
