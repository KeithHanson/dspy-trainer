- Optimization execution path: `backend/app/services.py::AppServices/run_optimization_job` loads train records from `dataset_id` or derives them from `source_run_plan_id`, optionally loads validation rows from `validation_dataset_id`, then calls `backend/app/executor/module_runner.py::run_bundle_optimization`.
- Dataset kind is strategy-dependent in `prepare_optimization_job_payload`: BootstrapFewShot and MIPROv2 require `demo` datasets; GEPA requires `feedback` datasets.
- Derived `demo` datasets include only eval tasks with numeric score >= threshold; target label comes from `label_payload` if present else accepted `prediction_payload`.
- Derived `feedback` datasets include both passing and failing eval tasks; each record carries `input`, original `label`, `prediction`, numeric `score`, `passed`, and textual `feedback`/`rationale` for GEPA reflection.
- There is no automatic train/validation split inside trainer optimization jobs. `validation_dataset_id` / `val_inputs` is the only separate val source. If absent, `run_bundle_optimization` falls back to building `valset` from the training records' `{input,label}` pairs.
- GEPA training examples are built as `dspy.Example(**input, **target_payload, optimization_feedback=feedback, label=label_payload or target_payload).with_inputs(...)`, where `target_payload` is mapped from label fields matching predictor outputs or from prediction outputs as fallback.
- Bundles can now declare `bundle.toml` `[optimization].target_output_fields = [..]`. When present, `run_bundle_optimization` uses that ordered field list as the optimization target surface instead of inferring all keys from prior prediction payloads. Runtime rejects declared fields that are not actual predictor outputs.
- GEPA compile metric returns `dspy.Prediction(score=..., feedback=...)`, combining record feedback with fresh judge rationale when both exist.
- After optimization, trainer always materializes bundle state, creates a follow-up eval plan/run against the optimized branch, and stores baseline vs optimized comparison summary on the job record.
